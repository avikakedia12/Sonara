# Core pipeline + API dependencies (see README's Setup section).
# Pinned to versions verified locally (Python 3.11) -- leaving these
# unpinned let pip's resolver backtrack across dozens of old releases
# trying to satisfy basic-pitch's transitive constraints, and crashed
# trying to build a years-old numpy from source once no prebuilt wheel
# existed for the interpreter version Render happened to use. Pinned,
# modern versions have prebuilt wheels for any current CPython, so pip
# fetches them directly instead of searching or compiling.
#
# numpy capped below 2.0: kept as a safety margin even after the
# tensorflow avoidance below -- verified via dry-run resolve that every
# other pin here (scipy, scikit-learn, numba) is happy with numpy<2.0
# and no backtracking/source builds are triggered, so there's no reason
# to risk the newer major version.
#
# basic-pitch is installed separately (see render.yaml's buildCommand)
# with --no-deps, deliberately skipping its normal tensorflow<2.15.1
# dependency (required unconditionally on Linux + Python>=3.11 per its
# own metadata). tensorflow<2.15.1 (Sept 2023) predates numpy 2.0 (June
# 2024) and its compiled extensions are ABI-incompatible with any numpy
# 2.x -- crashed every transcription request with "Unable to convert
# function return value to a Python type! The signature was ()
# -> handle" at model-load time on Render (Linux has no CoreML, which
# is what masks this locally on Mac -- see below). Even fixed, a numpy
# downgrade alone left tensorflow's ~400MB+ runtime import blowing past
# the 512MB starter-plan memory limit and crash-looping the instance.
# tflite-runtime is the fix basic-pitch itself documents for this case:
# a ~2MB purpose-built inference runtime, not a general ML framework,
# so importing it doesn't carry anywhere near that memory cost.
#
# --no-deps skips *all* of basic-pitch's dependencies, not just
# tensorflow -- pretty_midi and resampy (unlike scikit-learn/scipy,
# which librosa already pulls in) aren't required by anything else
# here, so they're listed explicitly below with basic-pitch's own
# version constraints (from its METADATA) to keep them satisfied.
librosa==0.11.0
numpy<2.0
music21==10.5.0
tflite-runtime==2.14.0; sys_platform == "linux"
pretty_midi>=0.2.9
resampy<0.4.3,>=0.2.2
mir_eval==0.8.2
fastapi==0.139.0
uvicorn==0.51.0
python-multipart==0.0.32
verovio==6.2.1
yt-dlp==2026.7.4
